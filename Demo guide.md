## Common Judge Questions

**Q: Where does the data come from?**
> "Live from WAQI's API (300+ Indian stations), NASA FIRMS satellite fire data updated every 3 hours, OpenStreetMap for the road network, and OpenWeatherMap for wind/temperature. All data sources are named in the Attribution Engine's confidence tooltip."

**Q: What makes the AI trustworthy?**
> "Every number in the system can be traced to its source. The Mayor Copilot shows which specific dataclass field backs every sentence. The Health Engine's assumptions expander cites the exact government survey and peer-reviewed papers it uses. No number is generated or estimated without documentation."

**Q: Could this actually be deployed?**
> "Each module is a stateless Python class with a clean input/output API. Wrapping them as FastAPI endpoints is a straightforward step. The Digital Twin already handles multi-city scenarios — you just change the population density and the intervention library. The architecture document explains each integration point."

**Q: Why didn't you use GPT-4 or Gemini for the Copilot?**
> "Because the requirement was *no fabrication* — and a generative LLM can paraphrase a number incorrectly. The Mayor Copilot's answers are 100% auditable: they're constructed by reading specific fields off already-computed dataclasses. We traded 'can answer anything' for 'every answer is provably correct.'"

**Q: What's the innovation over existing tools like CPCB Sameer or IQAir?**
> "Those tools show the AQI. AirTwin X shows the *cause* (attribution), the *remedy* (intervention ranking), the *predicted outcome* (digital twin), the *societal benefit* (health/economic engine), and the *decision rationale* (Mayor Copilot). The pipeline from sensor to decision is what's new."
